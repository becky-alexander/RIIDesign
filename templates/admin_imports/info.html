Regional Indicator Database

This site allows to input the necessary data for the Regional Indicators Inititiative website. For most of the inputs an entry needs to exist for every city and year. For example Minneapolis, 2012. This database allows to see all the existing records and a add new ones. The following is an explanation of each input:

 BASIC

There are basic inputs that are required for most of the indicators to work properly. Please double check that these are filled for the city you wish to alter.

  • Cities: It contains the pre-populated list of cities in Minnesota with their respective classification. A sort number must be given to a city for it to appear in the RII website. 
    • Classification: Cities are classified according to their type.
  • Demographic Data (Cities): The demographic data of cities is population and jobs. 
  • Demographic Data (Counties): The demographic data of counties. These are required for some indicators to work. 

 ENERGY

The following are the inputs required for the energy indicators. 

  • Electric Input: It contains the amount of electricity in kwh used by every city that has completed the program. It allows to specify the utility from which the electiricity is coming from and also whether there is wind electricity.
  • Natural Gas Input: It contains the amount of natural gas in therms used by every city that has completed the program. It allows to specify the utility from which the natural gas is coming. 
  • CDD & HDD: The Cooling Degree Days and Heating Degree Days are necessary to estimate the weather normalized numbers for energy. 
  • Other Fuels Input: This form allows to input the amount of other fuels. Other fuels are for example wood or coal used for direct heat generation. 
  • Energy Utilities: List of electric utilities. It allows to specify what type utility it is. 
  • Energy Utilities Emission Factors: Every energy utility has a corresponding emission factor. The emission factors allow to estimate the CO2 contribution of energy per unit.

 WATER

The following is the only input required for the water indicator.

  • Water Input: This form allows to input the amount of water in thousand gallons consumed at each city per year. This is the only input required for the water indicator to display.

 TRAVEL

  •  VMT: This contains the list of vehicles miles traveled per city per year. The list has been prepopulated, hence it is ideal not to alter the data. Use it more as a reference. 

 WASTE

  • Solid Waste Input: Contains the amout of solid waste for each county in Minnesota. From the county data the city data can be calculated. Enter the county data to estimate the solid wastte of the city.
  • Solid Waste Facilities: The list of waste processing facilities.
  • Solid Waste Facilities Emission Factors: When waste is taken to wste processing facilities it goes through certain process that release carbon dioxide. Depending on different variables including technology this emission factor might be different for each facility,, hence it needs to be specified. 

 COST

The cost factors allow to estimate the cost in USD per unit of energy, water, solid waste or travel.

  • Cost Factors: This form allows to input the cost per unit for each of the indicators.

 OTHER

There is other data that has to be inputed to have an accurate accounting of the greenhouse gas emission of a city. Some of this data is embeded in the database and does not need to be updated constantly, for example conversion factors. 

  • Wastewater Input: This allows to specify the amount of wastewater treated by city. 
  • Wastewater facilities: The list of wastewater facilities. 
  • Wastewater facilities (emission factors): Each wastewater facility has a different emission factors which needs to be specified each year.
